# 3. Core Methodology Architecture

The integrated Methodology Framework is organized around the Method Object.

```text
Scientific Taxonomy
        │
        ▼
Method Object
      ╱       ╲
     ▼         ▼
Evaluation   Implementation Mapping
     │         │
     └────┬────┘
          ▼
       Registry
          │
          ▼
    Method Library
          │
          ▼
 Pipeline Specification
          │
          ▼
 Coding Framework
```

Within this architecture:

* Taxonomy organizes methodologies.
* Method Objects represent methodologies.
* Evaluation determines scientific quality.
* Registry provides discovery and traceability.
* Method Library resolves reusable implementations.
* Pipeline Specification composes approved methodologies.
* Coding Framework executes implementations.

Each architectural component maintains an independent responsibility.

# 4. Core Architectural Principles

The Methodology Framework is governed by the following principles established throughout MP-000 to MP-006.

* Single Responsibility
* Single Source of Truth
* Progressive Refinement
* Scientific Methodology before Implementation
* Evidence before Recommendation
* Composition over Ownership
* Reuse before Reinvention
* Implementation Independence
* Traceability
* Long-term Extensibility

These principles collectively ensure a modular, reproducible, and maintainable methodology architecture.
